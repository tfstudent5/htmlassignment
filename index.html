Introduction

This course has been an interesting introduction to HTML (Hypertext Markup Language).HTML is a markup language that uses tags to structure content, which are then interpreted by a web browser. Although there are many different tags, they can broadly be divided into two main groups: structural and semantic. Structural tags form the basic framework or skeleton of a webpage, while semantic tags act as labelled containers that describe the meaning of the content they enclose.

While HTML has always included semantic elements such as headings and paragraphs, HTML5 introduced a range of new semantic tags designed to describe the structure and purpose of major sections of a webpage, such as <header>, <nav>, and <main>.

This course emphasised the importance of accessibility and demonstrated how semantic tags improve accessibility. For example, semantic tags can be interpreted by screen readers, making web pages more accessible to users who are blind or partially sighted. It was surprising to learn how many people live with disabilities, and failing to consider accessibility not only excludes a significant number of users but also undermines inclusive and responsible web design.

Structural and Semantic Tags

Structural Tags

Structural tags are used to organise the layout of a webpage and control how content is arranged on the screen. They help group sections of content together and create a clear framework for the page, but they do not describe what the content means. By using structural tags, developers can keep a webpage organised, easier to read, and easier to manage as it grows.

Examples of structural tags

  • <html> – the root element that contains the entire webpage
  • <head> – contains metadata and information about the document
  • <body> – contains all visible content on the webpage
  • <div> – groups blocks of content together
  • <span> – groups inline content within text
  • <br> – inserts a line break
  • <hr> – inserts a horizontal rule

Semantic Tags

Semantic tags are HTML elements that explain what the content on a webpage is and what purpose it serves. They help browsers and assistive technologies understand different sections of a page, rather than just displaying them. Using semantic tags makes webpages clearer, more accessible, and easier to understand.

Examples of Semantic Tags

  • <header> – Introductory content for a page or section, often containing headings or navigation.
  • <nav> – A section that contains primary navigation links.
  • <main> – The main, unique content of the page (only one per page).
  • <article> – Self-contained content that can stand alone or be reused, such as a blog post or news item.
  • <section> – A thematic grouping of related content within a page.
  • <aside> – Content indirectly related to the main content, such as sidebars or callouts.
  • <footer> – Closing or metadata content for a page or section, such as author info or links.

Accessibility

The W3C Validator

The W3C Markup Validator is not an accessibility tool; however, it supports accessibility by ensuring that HTML is valid and correctly structured, which is essential for assistive technologies to function reliably.

Another issue which was picked up by the W3C validator is setting the width to a percentage:

In the course we were told to set the width as a percentage so the image could be resized for different sized displays

<img src="images/w3cvalidator.jpg" width="50%">

However, this does not seem to be acceptable syntax, so I had to change it to pixels to avoid upsetting the w3c validator

<img src="images/w3cvalidator.jpg" width="800">

WAVE Web Accessibility Evaluation Tool

WAVE is a tool that checks web pages for accessibility problems and shows where changes are needed to make the site usable for people with disabilities. For example:

  • missing image alt text,
  • poor colour contrast,
  • incorrect heading structure

Other Things

HTML Character Entities

When creating this page, I needed to try and stop the browser from interpreting < and > as signifying the start and end of a tag. I did this using &lt; and &gt; which are character entities representing the less‑than symbol and greater than symbol. Such character entities are used when the character needs to be displayed as text rather than interpreted as part of an HTML tag. Other examples include:

  • &amp; - the ampersand & symbol
  • &quot; - for the "double quote" symbol
  • &nbsp; - for a non-breaking space
